Tipos de Grafos

No todos los grafos son iguales. Dependiendo de cómo sean sus aristas o nodos, tenemos diferentes tipos: cada uno modela unas situaciones mejor que otras.

Dirigido vs No Dirigido

La primera distinción fundamental en grafos es si las relaciones tienen dirección o no.

Grafo NO Dirigido — relación simétrica
Grafo Dirigido — relación asimétrica
Tipo Descripción Ejemplo real Neo4j
No Dirigido La relación va en ambos sentidos Amistad en Facebook, cables de red (a)-[:AMIGO]-(b)
Dirigido La relación tiene una dirección específica Seguir en Twitter, enviar email (a)-[:SIGUE]->(b)
⚠️
Ojo con Neo4j: En Neo4j todas las relaciones son dirigidas (tienen una flecha). Sin embargo, puedes hacer consultas ignorando la dirección con (a)-[:AMIGO]-(b) (sin flecha). El modelo de datos siempre almacena la dirección, pero las queries pueden ignorarla.

Grafos Ponderados

En un grafo ponderado, las aristas tienen un número asociado llamado peso. Este peso puede representar distancia, coste, tiempo, probabilidad, intensidad de la relación...

Red de carreteras con distancias (km)
cypher — Grafo ponderado en Neo4j 
// Crear una relación ponderada (con distancia)
CREATE (madrid:Ciudad {nombre: "Madrid"})
CREATE (toledo:Ciudad {nombre: "Toledo"})
CREATE (madrid)-[:CARRETERA {distancia_km: 45, tipo: "autopista"}]->(toledo)

// Encontrar la ruta más corta por peso
MATCH (inicio:Ciudad {nombre: "Madrid"}), (fin:Ciudad {nombre: "Alicante"})
CALL gds.shortestPath.dijkstra.stream('mi_grafo', {
  sourceNode: inicio,
  targetNode: fin,
  relationshipWeightProperty: 'distancia_km'
})
YIELD path, totalCost

Árboles

Un árbol es un tipo especial de grafo conexo y sin ciclos. Tiene una estructura jerárquica muy natural para representar jerarquías y clasificaciones.

Árbol — Estructura organizativa de empresa
🌳 Características de un Árbol
  • Sin ciclos (acíclico)
  • Exactamente un camino entre cualquier par de nodos
  • N nodos → N-1 aristas
  • Siempre conexo
📋 Ejemplos de Árboles
  • Jerarquía organizativa
  • Árbol de categorías de productos
  • Sistema de archivos
  • Árbol genealógico (simplificado)

Grafos Bipartitos

En un grafo bipartito, los nodos se dividen en dos grupos, y las aristas solo van de un grupo al otro: nunca dentro del mismo grupo. Son perfectos para modelar relaciones entre dos tipos de entidades.

Grafo bipartito — Clientes y Productos comprados
🛒
¿Por qué son útiles los grafos bipartitos? Son la base de los sistemas de recomendación. Si Ana y Luis han comprado iPhone y MacBook (mismos productos), el sistema puede recomendar a Ana los productos que ha comprado Luis pero ella no. Esto se llama collaborative filtering.

Multigrafos

Un multigrafo permite que existan múltiples aristas entre el mismo par de nodos. Neo4j soporta esto perfectamente: dos personas pueden estar conectadas por una relación AMIGO y también por TRABAJA_CON.

cypher — Multigrafo en Neo4j 
// Ana y Luis tienen DOS tipos de relación distintos
MATCH (ana:Persona {nombre: "Ana"}), (luis:Persona {nombre: "Luis"})

// Relación 1: Son amigos
CREATE (ana)-[:AMIGO {desde: 2018}]->(luis)

// Relación 2: También trabajan juntos
CREATE (ana)-[:TRABAJA_CON {proyecto: "Big Data"}]->(luis)

// Esto es un MULTIGRAFO: dos relaciones entre el mismo par de nodos

Resumen de Tipos

↔️

No Dirigido

Relaciones simétricas. Si A está conectado a B, entonces B está conectado a A. Ej: Facebook amigos.

➡️

Dirigido (Dígrafo)

Relaciones con sentido. A puede seguir a B sin que B le siga. Ej: Twitter follows.

⚖️

Ponderado

Las aristas tienen peso: distancia, coste, duración. Ej: Google Maps con tiempos de viaje.

🌳

Árbol

Grafo conexo sin ciclos. Jerarquía natural. N nodos → N-1 aristas. Ej: organigrama de empresa.

🎭

Bipartito

Dos grupos de nodos, aristas solo entre grupos. Base de los sistemas de recomendación. Ej: usuario-producto.

🔀

Multigrafo

Múltiples aristas entre el mismo par de nodos. Ej: dos personas conectadas por amistad Y trabajo.
