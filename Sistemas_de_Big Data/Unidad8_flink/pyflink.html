PyFlink — Flink con Python

Sí, Apache Flink tiene una API oficial para Python. Todo el poder de Flink con la simplicidad de Python.

1. ¿Qué es PyFlink?

PyFlink es la API oficial de Python para Apache Flink. Disponible desde Flink 1.9 (2019) y mejorada significativamente en versiones posteriores. Permite escribir jobs de Flink completamente en Python, sin necesidad de tocar Java.

🐍
¿Es igual que la versión de Java? Casi. PyFlink expone las mismas APIs (DataStream, Table API y SQL), pero hay algunas limitaciones: las operaciones stateful muy complejas y ciertas fuentes/sinks avanzados pueden requerir algo de Java/Scala como conector. Para el 90% de los casos de uso, Python es perfectamente suficiente.
✅ PyFlink es ideal para...
  • Data Scientists que conocen Python
  • Pipelines con lógica en pandas/numpy
  • Uso de modelos ML (scikit-learn, etc.)
  • Flink SQL (mismo SQL, sin Java)
  • Prototipos y proyectos de clase
⚠️ Limitaciones vs Java
  • Algo más de overhead (UDFs en Python se serializan)
  • Algunas fuentes/sinks solo en Java
  • Estado muy complejo más verboso
  • Documentación menos extensa
  • Ecosistema más joven

3. Tu primer job en PyFlink — Paso a Paso

Vamos a crear un job completo que lee palabras, las cuenta y muestra el resultado. El clásico "Word Count" del mundo de Big Data.

1
Crear el entorno de ejecución

Es el punto de partida de cualquier job Flink. Define si es streaming o batch.

2
Definir la fuente de datos (source)

De dónde vienen los datos: un fichero, Kafka, o una lista de elementos para pruebas.

3
Aplicar transformaciones

Usar map, filter, keyBy, reduce... para procesar los datos.

4
Escribir el resultado (sink)

Donde van los resultados: consola, fichero, base de datos, Kafka...

5
Ejecutar el job

Con env.execute(). Hasta este momento, Flink solo ha construido el plan, no ha ejecutado nada.

python — Word Count completo con PyFlink DataStream API 
from pyflink.datastream import StreamExecutionEnvironment
from pyflink.common.typeinfo import Types

# 1. Crear el entorno
env = StreamExecutionEnvironment.get_execution_environment()
env.set_parallelism(1)  # 1 instancia paralela (para pruebas)

# 2. Fuente: lista de frases de ejemplo
data = env.from_collection(
    collection=[
        "Apache Flink es genial",
        "Flink procesa streams en tiempo real",
        "Big Data con Apache Flink",
    ],
    type_info=Types.STRING()
)

# 3. Transformaciones: dividir en palabras y contar
word_count = (
    data
    .flat_map(                             # dividir cada frase en palabras
        lambda line, collector:
            [collector.collect((word, 1))
             for word in line.split()],
        output_type=Types.TUPLE([Types.STRING(), Types.INT()])
    )
    .key_by(lambda x: x[0])               # agrupar por palabra
    .reduce(lambda a, b: (a[0], a[1] + b[1]))  # sumar contadores
)

# 4. Sink: mostrar en consola
word_count.print()

# 5. Ejecutar
env.execute("Word Count con PyFlink")

# Salida esperada:
# ('Apache', 2)
# ('Flink', 3)
# ('es', 1)
# ('genial', 1)
# ('procesa', 1)
# ('streams', 1)  ...

4. PyFlink con Table API y SQL

La opción más sencilla para quien conoce SQL. Con la Table API puedes escribir queries SQL sobre streams en tiempo real exactamente igual que sobre tablas de una base de datos.

python — PyFlink con SQL: ventas por categoría en tiempo real 
from pyflink.table import EnvironmentSettings, TableEnvironment

# Crear Table Environment (streaming)
env_settings = EnvironmentSettings.in_streaming_mode()
t_env = TableEnvironment.create(env_settings)

# Crear tabla fuente desde Kafka
t_env.execute_sql("""
    CREATE TABLE ventas (
        producto   STRING,
        categoria  STRING,
        importe    DECIMAL(10, 2),
        ts         TIMESTAMP(3),
        WATERMARK FOR ts AS ts - INTERVAL '5' SECOND
    ) WITH (
        'connector' = 'kafka',
        'topic'     = 'ventas',
        'properties.bootstrap.servers' = 'localhost:9092',
        'format'    = 'json'
    )
""")

# Query SQL: ventas totales por categoría en ventanas de 1 hora
resultado = t_env.sql_query("""
    SELECT
        categoria,
        TUMBLE_START(ts, INTERVAL '1' HOUR) AS hora,
        SUM(importe)  AS total,
        COUNT(*)      AS num_ventas
    FROM ventas
    GROUP BY
        categoria,
        TUMBLE(ts, INTERVAL '1' HOUR)
""")

# Mostrar resultado
resultado.execute().print()

5. User Defined Functions (UDFs) en Python

Las UDFs permiten usar código Python personalizado dentro de queries SQL o Table API de Flink. ¡Incluso puedes usar librerías científicas como NumPy o scikit-learn!

python — UDF: calcular categoría de riesgo dentro de Flink SQL 
from pyflink.table.udf import udf
from pyflink.common import Row

# Definir la función Python como UDF
@udf(result_type='STRING')
def categoria_riesgo(importe: float, num_transacciones: int) -> str:
    """Clasifica el riesgo de un cliente basándose en su comportamiento."""
    if importe > 10000 or num_transacciones > 50:
        return "ALTO"
    elif importe > 1000 or num_transacciones > 10:
        return "MEDIO"
    else:
        return "BAJO"

# Registrar la UDF en el entorno
t_env.create_temporary_function("categoria_riesgo", categoria_riesgo)

# Usar la UDF en una query SQL ← como cualquier función SQL
t_env.execute_sql("""
    SELECT
        user_id,
        SUM(importe) AS total,
        COUNT(*)     AS num_tx,
        categoria_riesgo(SUM(importe), COUNT(*)) AS riesgo
    FROM transacciones
    GROUP BY user_id
""")

6. Comparativa: Java vs Python en Flink

Aspecto Java/Scala Python (PyFlink)
Rendimiento Máximo Alto (ligero overhead en UDFs)
Facilidad Media Alta (sintaxis Python)
APIs disponibles DataStream, Table, SQL, CEP, ML DataStream, Table, SQL (CEP limitado)
Librerías ML Solo Flink ML scikit-learn, pandas, numpy, etc.
Ideal para Producción, máximo rendimiento Data Science, prototipado, formación
En proyectos Big Data Estándar en grandes empresas Cada vez más adoptado
💡
Recomendación para clase: Para aprender Flink, PyFlink es perfecto. La sintaxis Python hace el código más legible y no te distrae con detalles de Java. Una vez que entiendes los conceptos (streams, ventanas, operadores), migrar a Java/Scala es trivial si el trabajo lo requiere.
