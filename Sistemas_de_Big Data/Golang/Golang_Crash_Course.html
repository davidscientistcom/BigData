Go Philosophy: Simplicity Over Features

  • No OOP – No classes, no inheritance, no polymorphism magic
  • No exceptions – Errors are values (explicit > implicit)
  • Minimal syntax – One way to do things (vs Python's "Zen")
  • Native concurrency – Goroutines (not threads or async/await)
  • Static typing + inference – Type safety without verbosity
  • Batteries included – stdlib covers 80% of needs
CRITICAL MINDSET SHIFT: Go rewards clarity over cleverness. If your Go code looks different from others, you're probably wrong. Embrace conventions.

Variables: The Go Way

// Explicit type (like Java) var edad int = 35 var nombre string = "David" // Type inference (like Python) edad := 35 // Only inside functions! nombre := "David" // No implicit conversions (unlike Python) var x int = 10 var y float64 = x // ❌ COMPILATION ERROR var y float64 = float64(x) // ✓ Explicit cast required
Python Java Go
age = 35 int age = 35; age := 35
Dynamic, runtime errors Static, compile-time safe Static + lightweight syntax

Zero Values: Go's Safety Net

Unlike Python (None), Go initializes every variable to a "zero value":

var i int // 0 var f float64 // 0.0 var s string // "" (empty string, not null) var b bool // false var p *int // nil (null for pointers) var m map[string]int // nil var sl []int // nil var arr [3]int // [0, 0, 0]

Why? No "undefined" state. A variable always has a valid value. Safer than Python's None-everywhere.

Pointers: Explicit Memory Management

Language Pointers Example
Python Hidden (everything is reference) x = [1, 2]; y = x (same object)
Java Hidden (for objects only) List x = new ArrayList(); List y = x;
C Explicit, manual management int *p = malloc(...); free(p);
Go Explicit, automatic cleanup p := &x; defer cleanup()
x := 42 p := &x // Get address (& operator) fmt.Println(*p) // Dereference (= 42) *p = 100 // Modify via pointer fmt.Println(x) // x is now 100

Functions: Multiple Returns (No Exceptions)

Python

def divide(a, b): if b == 0: raise ValueError("...") return a / b

Go

func divide(a, b float64) (float64, error) { if b == 0 { return 0, errors.New("...") } return a / b, nil }

Key difference: Go returns errors as values. No try/catch. Pattern: result, err := myFunc()

Idiom: Always check errors immediately. Panic is for programmer bugs, not runtime errors.

Structs: Composition Over Inheritance

Java (OOP)

class Animal { } class Dog extends Animal { } Dog dog = new Dog();

Go (Composition)

type Dog struct { } type Animal struct { Dog Dog // Embed }

No inheritance hierarchy. Structs are just data. Methods attached separately.

type User struct { Name string Email string } // Method (receiver type, then function name) func (u User) IsValid() bool { return u.Name != "" && u.Email != "" }

Interfaces: Implicit Satisfaction (Duck Typing)

// Define interface type Reader interface { Read(p []byte) (n int, err error) } // Satisfy interface (no explicit declaration!) type File struct { } func (f *File) Read(p []byte) (int, error) { // Implementation return 0, nil } // Use it var r Reader = &file // ✓ Works! File satisfies Reader

Why? Decoupling. Library defines Reader interface. Your code can implement it without importing library. Maximum flexibility.

Compare to Java: class File implements Reader {} – explicit coupling.

Error Handling: The Go Way

Errors are values, not exceptions. Explicit handling is required.

// Pattern 1: Immediate check file, err := os.Open("data.txt") if err != nil { log.Fatal(err) // Handle or panic } defer file.Close() // Resource cleanup // Pattern 2: Type assertion for custom errors if err != nil { if pathErr, ok := err.(*os.PathError); ok { fmt.Printf("Path error: %s\n", pathErr.Path) } }
idiom: Check errors early and often. No "error bubbling" like async/await. Every function with potential failure returns (result, error).

Concurrency: Goroutines (Not Threads)

Model Memory per unit Overhead Scaling
OS Threads (Python, Java) 1-2 MB High context switching 10,000 max
Go Goroutines ~2-8 KB Low multiplexing 1,000,000+ feasible
// Launch goroutine (like "thread" but lighter) go myFunction() // Or anonymous go func() { fmt.Println("Running concurrently") }()

Why Go dominates microservices: handle 10K concurrent requests with minimal resources.

Channels: Safe Goroutine Communication

Channels = typed "pipes" between goroutines (like message queues).

// Create channel ch := make(chan string) // Send (blocks until received) go func() { ch <- "Hello from goroutine" }() // Receive (blocks until sent) msg := <-ch fmt.Println(msg)
vs Threads: Python/Java need mutexes, atomic ops, complex sync. Go channels make safe communication idiomatic.

Key: "Share memory by communicating, don't communicate by sharing memory."

Project Structure: Go Modules

myproject/ ├── go.mod # Module definition (like package.json) ├── go.sum # Dependency checksums ├── main.go # Entry point (func main()) ├── utils/ │ ├── math.go # Package 'utils' │ └── strings.go └── models/ └── user.go # Package 'models'

go mod init github.com/david/myproject

Modules (go.sum + go.mod) handle versioning + lock files. No requirements.txt chaos like Python.

Defer: Guaranteed Cleanup

Python (try/finally)

try: file = open('data.txt') # use file finally: file.close()

Go (defer)

file, _ := os.Open('data.txt') defer file.Close() // use file // Close() ALWAYS runs

defer = LIFO stack. Last defer executes first. Perfect for resource cleanup.

Testing: Built-in, No Dependencies

package mypackage import "testing" func TestAdd(t *testing.T) { result := Add(2, 3) if result != 5 { t.Errorf("Expected 5, got %d", result) } }

go test ./... – Run all tests. No pytest, no unittest imports needed.

Testing is idiomatic in Go. Not optional like in Python shops.

Common Gotchas: Coming from Python/Java/C

  • No implicit type conversion – int + float64 = error. Cast explicitly.
  • := only in functions – Package-level needs var
  • Arrays vs Slices – [3]int is array (fixed). []int is slice (dynamic).
  • Nil is sneaky – nil != null. nil slice/map/channel are valid. Check carefully.
  • Loops are for only – No while. for i := 0; i < 10; i++
  • Blank identifier _ – _, err := func() to ignore return value.
