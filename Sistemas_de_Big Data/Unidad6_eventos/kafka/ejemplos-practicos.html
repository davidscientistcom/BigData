Ejemplos Prácticos

De lo simple a lo complejo: código que funciona

Ejemplo 1: Hola Kafka 👋

El ejemplo más simple: un productor envía 1 mensaje y un consumidor lo lee.

Productor Simple (Python)

#!/usr/bin/env python3
from kafka import KafkaProducer
import json

# Conectar
producer = KafkaProducer(
    bootstrap_servers=['localhost:9092'],
    value_serializer=lambda v: json.dumps(v).encode('utf-8')
)

# Enviar 1 mensaje
event = {
    'timestamp': '2024-01-15T10:30:00',
    'user': 'alice',
    'action': 'login'
}

future = producer.send('events', value=event)
record_metadata = future.get(timeout=10)

print(f"✅ Enviado a partition {record_metadata.partition} con offset {record_metadata.offset}")
producer.close()

Consumidor Simple (Python)

#!/usr/bin/env python3
from kafka import KafkaConsumer
import json

# Conectar
consumer = KafkaConsumer(
    'events',
    bootstrap_servers=['localhost:9092'],
    auto_offset_reset='earliest',
    value_deserializer=lambda m: json.loads(m.decode('utf-8'))
)

# Leer mensajes
for message in consumer:
    print(f"📨 Evento: {message.value}")
    print(f"Offset: {message.offset}")
Cómo ejecutar:
  1. Levanta Kafka con Docker (ver sección Docker Compose)
  2. Terminal 1: python productor.py
  3. Terminal 2: python consumidor.py
  4. Debería ver el evento en terminal 2

Ejemplo 2: Múltiples Mensajes en Bucle 🔄

El productor envía mensajes continuamente. El consumidor sigue leyendo.

Productor Continuo (Python)

#!/usr/bin/env python3
from kafka import KafkaProducer
import json
import time
import random

producer = KafkaProducer(
    bootstrap_servers=['localhost:9092'],
    value_serializer=lambda v: json.dumps(v).encode('utf-8'),
    acks='all',  # Esperar confirmación
    retries=3
)

users = ['alice', 'bob', 'charlie', 'david']
actions = ['login', 'purchase', 'logout', 'view_product']

for i in range(100):  # Enviar 100 mensajes
    event = {
        'id': i,
        'user': random.choice(users),
        'action': random.choice(actions),
        'timestamp': time.time()
    }
    
    producer.send('events', 
                  key=event['user'].encode(),  # Key = user (para particionamiento)
                  value=event)
    
    print(f"📤 Enviado evento {i}: {event['user']} → {event['action']}")
    time.sleep(1)

producer.flush()
producer.close()

Consumidor Continuo (Python)

#!/usr/bin/env python3
from kafka import KafkaConsumer
import json

consumer = KafkaConsumer(
    'events',
    bootstrap_servers=['localhost:9092'],
    group_id='analytics-group',
    auto_offset_reset='earliest',
    value_deserializer=lambda m: json.loads(m.decode('utf-8')),
    enable_auto_commit=True,
    auto_commit_interval_ms=5000
)

print("Escuchando eventos...")
count = 0

for message in consumer:
    count += 1
    event = message.value
    print(f"[{count}] 📨 {event['user']} → {event['action']} "
          f"(offset: {message.offset}, partition: {message.partition})")
🔍 Observa:
  • key=event['user'].encode() → Todos los eventos del mismo usuario van a la misma partición
  • acks='all' → El productor espera confirmación. Más seguro.
  • enable_auto_commit=True → El consumidor guarda automaticamente su offset

Ejemplo 3: Múltiples Consumidores (Paralelismo) 🚀

3 consumidores procesando el MISMO topic en paralelo. Kafka distribuye particiones automáticamente.

💡

Setup:

Asume que ya creaste el topic con al menos 3 particiones (haz esto primero):

kafka-topics --create --topic events \
  --partitions 3 \
  --replication-factor 1 \
  --bootstrap-server localhost:9092

Productor (igual al anterior) 📤

Usa el mismo `productor.py` del ejemplo 2

Consumidor 1 (Python) 👤

#!/usr/bin/env python3
from kafka import KafkaConsumer
import json

consumer = KafkaConsumer(
    'events',
    bootstrap_servers=['localhost:9092'],
    group_id='analytics-group',  # MISMO grupo en los 3 consumidores
    auto_offset_reset='earliest',
    value_deserializer=lambda m: json.loads(m.decode('utf-8'))
)

import socket
hostname = socket.gethostname()

for message in consumer:
    event = message.value
    print(f"[Worker {hostname}] {event['user']} → {event['action']} "
          f"(partition: {message.partition})")

Ejecutar los 3 consumidores 🔀

# Terminal 1
python consumidor.py

# Terminal 2
python consumidor.py

# Terminal 3
python consumidor.py

# Terminal 4: Productor
python productor.py
🔍 Observa:
  • Todos tienen el mismo group_id='analytics-group'
  • Kafka automáticamente asigna: Worker 1 → Partition 0, Worker 2 → Partition 1, Worker 3 → Partition 2
  • Cada worker procesa su partición en paralelo 🔥
  • Si 1 worker crashea, Kafka rebalanceo y reasigna particiones

Ejemplo 4: Filtrado en Tiempo Real & Guardar en BD 💾

Consumidor que procesa eventos, filtra, y guarda en SQLite.

Consumidor Inteligente (Python)

#!/usr/bin/env python3
from kafka import KafkaConsumer
import json
import sqlite3

# Conexión BD
conn = sqlite3.connect('events.db')
cursor = conn.cursor()

# Crear tabla
cursor.execute('''
    CREATE TABLE IF NOT EXISTS purchases (
        id INTEGER PRIMARY KEY,
        user TEXT,
        timestamp REAL,
        action TEXT
    )
''')
conn.commit()

# Consumidor
consumer = KafkaConsumer(
    'events',
    bootstrap_servers=['localhost:9092'],
    group_id='db-group',
    auto_offset_reset='earliest',
    value_deserializer=lambda m: json.loads(m.decode('utf-8')),
    enable_auto_commit=False  # Manual commit
)

for message in consumer:
    event = message.value
    
    # Filtrar: solo guardar "purchase"
    if event['action'] == 'purchase':
        try:
            cursor.execute(
                'INSERT INTO purchases (user, timestamp, action) VALUES (?, ?, ?)',
                (event['user'], event['timestamp'], event['action'])
            )
            conn.commit()
            print(f"✅ Guardado: {event['user']} compró")
            
            # Commit offset SOLO después de guardar
            consumer.commit()
            
        except Exception as e:
            print(f"❌ Error: {e}")
            conn.rollback()

conn.close()
🔍 Observa:
  • enable_auto_commit=False → Control manual. Garantiza exactly-once
  • Solo guardamos si action=='purchase'. Filtrado en tiempo real.
  • Hacemos consumer.commit() DESPUÉS de guardar en BD
  • Si crashea entre guardar y commit, se reinicia desde el mismo mensaje (idempotente)

Ejemplo 5: Dead Letter Queue (Manejo de Errores) 💀

Mensajes que fallan se envían a otro topic (DLQ).

Consumidor con DLQ (Python)

#!/usr/bin/env python3
from kafka import KafkaConsumer, KafkaProducer
import json

# Productor para DLQ
dlq_producer = KafkaProducer(
    bootstrap_servers=['localhost:9092'],
    value_serializer=lambda v: json.dumps(v).encode('utf-8')
)

# Consumidor
consumer = KafkaConsumer(
    'events',
    bootstrap_servers=['localhost:9092'],
    group_id='dlq-group',
    auto_offset_reset='earliest',
    value_deserializer=lambda m: json.loads(m.decode('utf-8'))
)

for message in consumer:
    event = message.value
    
    try:
        # Intentar procesar
        if 'user' not in event:
            raise ValueError("Missing 'user' field")
        
        # Procesar...
        print(f"✅ Procesado: {event}")
        
    except Exception as e:
        # Enviar a DLQ
        error_event = {
            'original_event': event,
            'error': str(e),
            'timestamp': message.timestamp,
            'partition': message.partition,
            'offset': message.offset
        }
        
        dlq_producer.send('events-dlq', value=error_event)
        print(f"❌ Erro en evento {message.offset}, enviado a DLQ")

dlq_producer.close()
🔍 Usabilidad:
  • Crea 2 topics: events y events-dlq
  • Si un evento falla, se envía a events-dlq con metadata del error
  • Posteriormente, analiza DLQ para debugging 🐛

Comparación de Ejemplos 📊

Ejemplo Complejidad Caso de Uso Garantías
1: Hola Kafka ⭐ Básico Learning, PoC At-most-once
2: Múltiples ⭐⭐ Simple Logs, telemetría At-least-once
3: Paralelismo ⭐⭐⭐ Medio Procesamiento escalado At-least-once
4: BD + Filtrado ⭐⭐⭐⭐ Avanzado ETL, warehouse Exactly-once
5: DLQ ⭐⭐⭐⭐ Avanzado Producción, resiliencia Exactly-once + recovery
