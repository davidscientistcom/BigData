Qué incluye:

  • Zookeeper: Coordinador del cluster
  • 3 Brokers Kafka: El cluster (puedes escalar a más)
  • Networking: Todos conectados en la red interna

🚀 docker-compose.yml (Modo KRaft - Recomendado)

✅ Sin ZooKeeper - Kafka 3.3+ coordina por sí solo

version: '3.8'

services:
  kafka1:
    image: confluentinc/cp-kafka:7.5.0
    environment:
      KAFKA_BROKER_ID: 1
      KAFKA_LISTENER_SECURITY_PROTOCOL_MAP: CONTROLLER:PLAINTEXT,PLAINTEXT:PLAINTEXT,PLAINTEXT_HOST:PLAINTEXT
      KAFKA_ADVERTISED_LISTENERS: PLAINTEXT://kafka1:29092,PLAINTEXT_HOST://localhost:9092
      KAFKA_PROCESS_ROLES: broker,controller
      KAFKA_NODE_ID: 1
      KAFKA_CONTROLLER_QUORUM_VOTERS: 1@kafka1:29093,2@kafka2:29093,3@kafka3:29093
      KAFKA_CONTROLLER_LISTENER_NAMES: CONTROLLER
      KAFKA_LOG_DIRS: /tmp/kraft-combined-logs
      CLUSTER_ID: IoTab_g3S9qt37B55K_H6w
    ports:
      - "9092:9092"
    networks:
      - kafka-network

  kafka2:
    image: confluentinc/cp-kafka:7.5.0
    environment:
      KAFKA_BROKER_ID: 2
      KAFKA_LISTENER_SECURITY_PROTOCOL_MAP: CONTROLLER:PLAINTEXT,PLAINTEXT:PLAINTEXT,PLAINTEXT_HOST:PLAINTEXT
      KAFKA_ADVERTISED_LISTENERS: PLAINTEXT://kafka2:29092,PLAINTEXT_HOST://localhost:9093
      KAFKA_PROCESS_ROLES: broker,controller
      KAFKA_NODE_ID: 2
      KAFKA_CONTROLLER_QUORUM_VOTERS: 1@kafka1:29093,2@kafka2:29093,3@kafka3:29093
      KAFKA_CONTROLLER_LISTENER_NAMES: CONTROLLER
      KAFKA_LOG_DIRS: /tmp/kraft-combined-logs

      - kafka-network

  kafka3:
    image: confluentinc/cp-kafka:7.5.0
    environment:
      KAFKA_BROKER_ID: 3
      KAFKA_LISTENER_SECURITY_PROTOCOL_MAP: CONTROLLER:PLAINTEXT,PLAINTEXT:PLAINTEXT,PLAINTEXT_HOST:PLAINTEXT
      KAFKA_ADVERTISED_LISTENERS: PLAINTEXT://kafka3:29092,PLAINTEXT_HOST://localhost:9094
      KAFKA_PROCESS_ROLES: broker,controller
      KAFKA_NODE_ID: 3
      KAFKA_CONTROLLER_QUORUM_VOTERS: 1@kafka1:29093,2@kafka2:29093,3@kafka3:29093
      KAFKA_CONTROLLER_LISTENER_NAMES: CONTROLLER
      KAFKA_LOG_DIRS: /tmp/kraft-combined-logs
      CLUSTER_ID: IoTab_g3S9qt37B55K_H6w
    ports:
      - "9094:9094"
    networks:
      - kafka-network

networks:
  kafka-network:
    driver: bridge
¿Qué es todo esto?
  • KAFKA_PROCESS_ROLES: Cada broker es "broker" y "controller" (coordinación integrada)
  • KAFKA_NODE_ID: ID único del nodo (1, 2, 3)
  • KAFKA_CONTROLLER_QUORUM_VOTERS: Todos los nodos que pueden participar en elecciones
  • CLUSTER_ID: ID único del cluster (KRaft lo requiere, ejemplo dado)
  • Sin ZOOKEEPER_CONNECT: ¡Ya no necesitas ZooKeeper externo!

Comandos Útiles 🔨

Levantar el cluster

docker-compose up -d

Ver logs

docker-compose logs -f kafka1

Crear un topic (3 particiones, factor replicación 2)

docker exec kafka1 kafka-topics --create \
  --topic events \
  --partitions 3 \
  --replication-factor 2 \
  --bootstrap-server localhost:9092

Ver todos los topics

docker exec kafka1 kafka-topics --list \
  --bootstrap-server localhost:9092

Describir un topic

docker exec kafka1 kafka-topics --describe \
  --topic events \
  --bootstrap-server localhost:9092

Ver Offsets de un Consumer Group

docker exec kafka1 kafka-consumer-groups --describe \
  --group analytics-group \
  --bootstrap-server localhost:9092

Resetear offset del grupo

docker exec kafka1 kafka-consumer-groups --reset-offsets \
  --group analytics-group \
  --topic events \
  --to-earliest \
  --execute \
  --bootstrap-server localhost:9092

Detener el cluster

docker-compose down

Detener Y limpiar datos

docker-compose down -v

Testing: Verifica que Todo Funciona ✅

test_connection.py

#!/usr/bin/env python3
from kafka import KafkaProducer, KafkaConsumer
import json
import time

print("🔍 Verificando conexión a Kafka...")

try:
    # Test productor
    producer = KafkaProducer(
        bootstrap_servers=['localhost:9092'],
        value_serializer=lambda v: json.dumps(v).encode('utf-8'),
        request_timeout_ms=5000
    )
    producer.send('test', value={'msg': 'hello'})
    producer.close()
    print("✅ Productor OK")

    time.sleep(1)

    # Test consumidor
    consumer = KafkaConsumer(
        'test',
        bootstrap_servers=['localhost:9092'],
        auto_offset_reset='earliest',
        value_deserializer=lambda m: json.loads(m.decode('utf-8')),
        consumer_timeout_ms=5000
    )
    
    for message in consumer:
        print(f"✅ Consumidor OK - Leído: {message.value}")
        break
    
    consumer.close()
    print("\n🎉 ¡TODO FUNCIONA!")

except Exception as e:
    print(f"❌ Error: {e}")
    print("Verifica que:")
    print("1. Docker está corriendo: docker-compose up -d")
    print("2. Los brokers están listos: espera 10-15 segundos")
    print("3. El tema 'test' existe: docker exec kafka1 kafka-topics...")
Ejecutar test: 
python test_connection.py

Troubleshooting 🐛

❌ "Connection refused"

Causa: Kafka aún no está listo.

Solución:

docker-compose logs kafka1
# Espera hasta ver "started (org.apache.kafka.server.KafkaServer)"

❌ "Topic does not exist"

Causa: No creaste el topic.

Solución:

docker exec kafka1 kafka-topics --create --topic tu-topic \
  --partitions 1 --replication-factor 1 --bootstrap-server localhost:9092

❌ "Address already in use"

Causa: Ya tienes algo en los puertos 9092-9094.

Solución:

docker-compose down
# Luego modifica los puertos en docker-compose.yml

❌ Consumidor no lee nada

Causa: El offset está adelantado.

Solución:

docker exec kafka1 kafka-consumer-groups --reset-offsets \
  --group my-group \
  --topic my-topic \
  --to-earliest \
  --execute \
  --bootstrap-server localhost:9092

❌ Los contenedores mueren al iniciar

Causa: Falta de memoria o errores de config.

Solución:

docker-compose down -v
docker system prune
docker-compose up -d

# Ver logs detallados
docker-compose logs

Monitoreo y UI (Opcional) 📊

Kafka UI es una interfaz web para ver topics, mensajes, consumer groups en tiempo real.

docker-compose.yml con Kafka UI

# Agrega esto al archivo docker-compose.yml:

  kafka-ui:
    image: provectuslabs/kafka-ui:latest
    depends_on:
      - kafka1
      - kafka2
      - kafka3
    ports:
      - "8080:8080"
    environment:
      KAFKA_CLUSTERS_0_NAME: kafka-local
      KAFKA_CLUSTERS_0_BOOTSTRAPSERVERS: kafka1:29092,kafka2:29093,kafka3:29094
      KAFKA_CLUSTERS_0_ZOOKEEPER: zookeeper:2181
    networks:
      - kafka-network
🌐

Después de agregar:

docker-compose up -d

Abre: http://localhost:8080

Verás: Topics, Particiones, Mensajes, Consumer Groups, etc.
