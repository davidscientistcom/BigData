Laboratorio 01

🌱 Práctica: Hola Mundo Kafka

Entorno desde cero construyendo tu primer Producer y Consumer asíncrono para enviar eventos en una partición. Ideal para perder el miedo.

Terminal: Preparando el terreno
Paso 1: Crear el topic base

Primero, entraremos a nuestro contenedor de Docker y usaremos la CLI nativa para crear un topic ridículamente simple: 1 partición, 1 réplica.

# Entramos al servidor de Kafka
docker exec -it kafka bash

# Dentro de Kafka, creamos el topic
kafka-topics --bootstrap-server localhost:9092 \
             --create \
             --topic clicks_web \
             --partitions 1 \
             --replication-factor 1
✅ ¡Listo! Con esto Kafka ya tiene asignado un bloque en disco para empezar a recibir bytes infinitamente en el canal clicks_web.
El Producer: Disparando el Evento
Paso 2: Código para Enviar (Push)

El trabajo del Producer es simplemente serializar sus datos (los convertimos a array de bytes, ej. UTF-8) y dispararlos al topic. Compararemos cómo se ve en Python y en Java.

🐍 Python (confluent_kafka)

from confluent_kafka import Producer
import json

# 1. Configurar conexión
conf = {'bootstrap.servers': 'localhost:9094'}
producer = Producer(conf)

# 2. Preparar el mensaje (JSON)
evento = {"user": "Alice", "accion": "click"}
valor_bytes = json.dumps(evento).encode('utf-8')

# 3. Disparar asíncronamente
producer.produce('clicks_web', value=valor_bytes)

# 4. Asegurarnos de vaciar la cola antes de salir
producer.flush()
print("Mensaje enviado con éxito!")

☕ Java (KafkaProducer)

import org.apache.kafka.clients.producer.*;
import java.util.Properties;

// 1. Configurar conexión y serializadores
Properties props = new Properties();
props.put("bootstrap.servers", "localhost:9094");
props.put("key.serializer", "org.apache.kafka.common.serialization.StringSerializer");
props.put("value.serializer", "org.apache.kafka.common.serialization.StringSerializer");

KafkaProducer<String, String> producer = new KafkaProducer<>(props);

// 2. Crear récord y enviarlo
String evento = "{\"user\": \"Alice\", \"accion\": \"click\"}";
ProducerRecord<String, String> record = new ProducerRecord<>("clicks_web", evento);

producer.send(record); // Asíncrono
producer.close();
System.out.println("Mensaje enviado con éxito!");
El Consumer: Leyendo en Bucle Infinito
Paso 3: Código para Recibir (Pull)

El Consumer se suscribe al canal y entra en un bucle infinito (while True). Pide mensajes a Kafka activamente. Si no hay nada, espera callado.

🐍 Python (Consumer)

from confluent_kafka import Consumer

# 1. Configurar grupo
conf = {
    'bootstrap.servers': 'localhost:9094',
    'group.id': 'grupo-python-1',
    'auto.offset.reset': 'earliest' # Leer desde cero la 1° vez
}
consumer = Consumer(conf)

# 2. Suscribirse
consumer.subscribe(['clicks_web'])

# 3. Bucle infinito (El Motor)
print("Escuchando mensajes...")
while True:
    msg = consumer.poll(1.0) # Bloquea máximo 1 seg esperando
    
    if msg is None:
        continue
    if msg.error():
        print(f"Error: {msg.error()}")
        continue
        
    print(f"Recibido: {msg.value().decode('utf-8')}")

☕ Java (KafkaConsumer)

import org.apache.kafka.clients.consumer.*;
import java.time.Duration;
import java.util.*;

// 1. Configurar grupo
Properties p = new Properties();
p.put("bootstrap.servers", "localhost:9094");
p.put("group.id", "grupo-java-1");
p.put("auto.offset.reset", "earliest");
p.put("key.deserializer", "org.apache.kafka.common.serialization.StringDeserializer");
p.put("value.deserializer", "org.apache.kafka.common.serialization.StringDeserializer");

KafkaConsumer<String, String> consumer = new KafkaConsumer<>(p);

// 2. Suscribirse
consumer.subscribe(Arrays.asList("clicks_web"));

// 3. Bucle infinito
System.out.println("Escuchando mensajes...");
while (true) {
    ConsumerRecords<String, String> records = consumer.poll(Duration.ofMillis(1000));
    for (ConsumerRecord<String, String> record : records) {
        System.out.printf("Recibido: %s%n", record.value());
    }
}
