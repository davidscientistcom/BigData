Laboratorio 02

⚖️ Práctica: Escalabilidad y Grupos

Divide y vencerás. Vamos a ver en código cómo funciona el Hash-routing por llave en 3 particiones y un Consumer Group de 3 procesos leyendo en paralelo.

Terminal: Preparando el Topic a gran escala
Paso 1: Crear el topic con N particiones

Para paralelizar el procesamiento entre 3 maquinitas (Consumers) y garantizar que la información se esparza uniformemente, necesitamos al menos 3 carriles lógicos en Kafka.

# Desde dentro del contenedor kafka
kafka-topics --bootstrap-server localhost:9092 \
             --create \
             --topic transacciones \
             --partitions 3 \
             --replication-factor 1
✅ Tienes 3 contenedores lógicos [Partición-0, Partición-1, Partición-2]. Listo para repartir la carga.
Enrutamiento Hash en el Producer
Paso 2: Enviar con `KEY` para respetar el orden

Las Keys son vitales en sistemas paralelos. Si mandas eventos de "Juan" sin llave, Kafka los lanza a particiones aleatorias (Round-Robin). Si le pones una llave, todos los eventos de "Juan" irán siempre a la misma partición y sus eventos jamás se desordenarán.

🐍 Python (Producer con Key)

from confluent_kafka import Producer
import json
import time

producer = Producer({'bootstrap.servers': 'localhost:9094'})

eventos = [
    {"user": "u_1", "monto": 50},  # Juan
    {"user": "u_2", "monto": 10},  # Elena
    {"user": "u_1", "monto": 25}   # Juan de nuevo
]

for ev in eventos:
    # LA CLAVE! El hash-routing garantiza que u_1 va siempre 
    # a la misma partición
    llave = ev["user"].encode('utf-8')
    valor = json.dumps(ev).encode('utf-8')

    producer.produce('transacciones', key=llave, value=valor)
    time.sleep(0.5)

producer.flush()

☕ Java (KafkaProducer con Key)

import org.apache.kafka.clients.producer.*;
import java.util.Properties;

Properties p = new Properties();
p.put("bootstrap.servers", "localhost:9094");
p.put("key.serializer", "org.apache.kafka.common.serialization.StringSerializer");
p.put("value.serializer", "org.apache.kafka.common.serialization.StringSerializer");

KafkaProducer<String, String> producer = new KafkaProducer<>(p);

// Nota cómo al ProducerRecord le inyectamos 3 parámetros en vez de 2
// ("topic", "key", "value")
producer.send(new ProducerRecord<>("transacciones", "u_1", "{\"monto\": 50}"));
producer.send(new ProducerRecord<>("transacciones", "u_2", "{\"monto\": 10}"));
producer.send(new ProducerRecord<>("transacciones", "u_1", "{\"monto\": 25}"));

producer.close();
Consumer Group: El Reparto Automático
Paso 3: Un Grupo, múltiples workers

El poder de Kafka radica en su orquestador de Grupos. Abre 3 consolas distintas y ejecuta el mismo código exacto en las tres al mismo tiempo. Al leer que los tres tienen el mismo group.id="grupo_transacciones", el clúster les asignará a cada uno una partición exclusiva (1 Consumer -> 1 Partición). ¡Paralelismo automático sin usar websockets ni balanceadores de carga!

🐍 Python (Workers)

from confluent_kafka import Consumer

# Ejecuta este script 3 veces en 3 terminales distintas
conf = {
    'bootstrap.servers': 'localhost:9094',
    'group.id': 'grupo_transacciones', # <- El secreto
    'auto.offset.reset': 'earliest'
}
consumer = Consumer(conf)
consumer.subscribe(['transacciones'])

while True:
    msg = consumer.poll(1.0)
    if not msg or msg.error():
        continue
    # Mostramos qué partición le tocó a este Worker (0, 1 o 2)
    p = msg.partition()
    print(f"[Worker P-{p}] Leyó llave: {msg.key()} -> {msg.value()}")

☕ Java (Workers)

import org.apache.kafka.clients.consumer.*;
import java.time.Duration;
import java.util.*;

Properties p = new Properties();
p.put("bootstrap.servers", "localhost:9094");
p.put("group.id", "grupo_transacciones"); // <- El secreto
// Deserializadores...

KafkaConsumer<String, String> worker = new KafkaConsumer<>(p);
worker.subscribe(Arrays.asList("transacciones"));

while (true) {
    ConsumerRecords<String, String> recs = worker.poll(Duration.ofMillis(100));
    for (ConsumerRecord<String, String> r : recs) {
        System.out.printf("[Worker P-%d] Leyó llave: %s -> %s%n", 
            r.partition(), r.key(), r.value());
    }
}
💥 Prueba de Rebalanceo Cooperativo: Con los 3 procesos (Workers) encendidos, cancela abruptamente uno apretando Ctrl+C.

Mágicamente verás cómo Kafka detecta que ese integrante murió, y en cosa de un segundo los otros 2 procesos que siguen vivos adoptarán la Partición huérfana para que los mensajes sigan siendo procesados. ¡Alta resiliencia gratuita!
