Práctica Binarización: Confluent Avro

Implementación desde cero de un Producer y Consumer utilizando el Schema Registry de Confluent para inyectar payloads binarios ultra densos.

1. Preparación del Entorno

A diferencia del JSON simple que viene integrado en todos los lenguajes, utilizar Avro requiere comunicarse con el Schema Registry por HTTP, compilar el esquema y serializar bytes.

# 1. Asegúrate de tener levantado el entorno Docker con el Registry (puerto 8081)
docker-compose up -d

# 2. Instala la librería de Confluent con el extra [avro]
pip install "confluent-kafka[avro]"

2. Definir el Contrato: El Archivo `.avsc`

Debemos crear un archivo físico en nuestro proyecto que defina estrictamente cómo son los datos. Crea un archivo llamado user_click.avsc en la raíz de tu proyecto.

user_click.avscJSON
{
  "namespace": "com.bootcamp.avro",
  "type": "record",
  "name": "UserClick",
  "fields": [
    {"name": "user_id", "type": "int"},
    {"name": "button_id", "type": "string"},
    {"name": "timestamp_ms", "type": "long", "logicalType": "timestamp-millis"}
  ]
}

3. El Productor (AvroProducer)

El Productor en Python seguirá utilizando un diccionario normal (tipo Pydantic/JSON), pero usará la clase AvroSerializer. Esta clase interceptará el diccionario, lo validará contra el .avsc, contactará al Schema Registry para registrar el ID, y finalmente lo aplastará a bytes antes de inyectarlo en Kafka.

producer_avro.pyPython
import time
from confluent_kafka import SerializingProducer
from confluent_kafka.schema_registry import SchemaRegistryClient
from confluent_kafka.schema_registry.avro import AvroSerializer

# 1. Leer el archivo local del esquema
with open("user_click.avsc", "r") as f:
    schema_str = f.read()

# 2. Conectar al Schema Registry (Levantado por tu Docker)
schema_registry_conf = {'url': 'http://localhost:8081'}
schema_registry_client = SchemaRegistryClient(schema_registry_conf)

# 3. Crear el serializador inyector
avro_serializer = AvroSerializer(schema_registry_client, schema_str)

# 4. Configurar el Productor (¡Ojo! Usamos SerializingProducer)
producer_conf = {
    'bootstrap.servers': 'localhost:9092',
    'value.serializer': avro_serializer  # Magia Ocurre Aquí
}
producer = SerializingProducer(producer_conf)

# 5. Producir el evento (Parece un Dict, pero viajará en Binario)
event = {
    "user_id": 104,
    "button_id": "COMPRAR_AHORA",
    "timestamp_ms": int(time.time() * 1000)
}

producer.produce(topic='clicks_avro', value=event)
producer.flush()

print("✅ Mensaje Avro enviado exitosamente con Magic Byte 0x00")
¿Qué pasó por debajo? El AvroSerializer contactó a http://localhost:8081. El Registry le dijo: "Este esquema es el ID 1". El Productor cogió el número 104, el string "COMPRAR_AHORA" y el tiempo, purgó las claves ("user_id", etc.), concatenó el Magic Byte [0x00 | ID 1 | Bytes Puros] y lo disparó a la red.

4. El Consumidor (AvroConsumer)

El consumidor no necesita tener el fichero .avsc en su disco. Automáticamente leerá el ID 1 del paquete binario y le pedirá la estructura al Registry en tiempo real.

consumer_avro.pyPython
from confluent_kafka import DeserializingConsumer
from confluent_kafka.schema_registry import SchemaRegistryClient
from confluent_kafka.schema_registry.avro import AvroDeserializer

# 1. Cliente Schema Registry
sr_client = SchemaRegistryClient({'url': 'http://localhost:8081'})

# 2. Deserializador (Se bajará la estructura automáticamente)
avro_deserializer = AvroDeserializer(sr_client)

# 3. Configurar Consumer (DeserializingConsumer)
consumer_conf = {
    'bootstrap.servers': 'localhost:9092',
    'group.id': 'analytics_group',
    'auto.offset.reset': 'earliest',
    'value.deserializer': avro_deserializer # Magia Inversa
}
consumer = DeserializingConsumer(consumer_conf)
consumer.subscribe(['clicks_avro'])

print("🎧 Escuchando mensajes binarios...")

while True:
    try:
        msg = consumer.poll(1.0)
        if msg is None:
            continue
        
        # El Deserializer ya convirtió los bytes de nuevo a Dict de Python
        evento_json = msg.value() 
        print(f"Recibido Click de Usuario: {evento_json['user_id']} en {evento_json['button_id']}")
        
    except KeyboardInterrupt:
        break

consumer.close()
Fallo Común de Producción: Tolerancia a Caídas.
A simple vista parece que el Consumer fallará si el contenedor de Docker del Schema Registry se cae. En realidad, el AvroDeserializer guarda los esquemas cacheados en la memoria RAM del contenedor de Python. Si el Registry se cae 10 minutos, los Productor y los Consumidores seguirán funcionando sin inmutarse siempre y cuando no se registre un "Schema Nuevo".
