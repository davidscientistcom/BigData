Casos de Uso Reales

Arquitecturas completas: detección de fraude, análisis de logs, IoT, e-commerce en tiempo real. Así usan Flink las grandes empresas.

1. Detección de Fraude Bancario en Tiempo Real

Este es el caso de uso más popular de Flink. Los bancos procesan millones de transacciones al día y necesitan detectar fraude antes de que el dinero salga de la cuenta. Cada segundo cuenta.

🏦
Empresas que lo usan: Alibaba (AliPay), PayPal, ING Bank, Banco Santander. El sistema de Alibaba procesa más de 1 millón de transacciones por segundo durante el Single's Day usando Apache Flink.

Señales de fraude que detecta Flink:

  • Velocidad inusual: Más de 5 transacciones en 1 minuto
  • Importe atípico: Transacción muy superior a la media histórica del usuario
  • Localización imposible: Compra en Madrid y 10 minutos después en Tokyo
  • Patrón de prueba: Varias transacciones pequeñas rápidas (testeo de tarjeta robada)
Arquitectura: Sistema de detección de fraude con Flink
java — Detección: más de 3 transacciones pequeñas seguidas (card testing) 
// Detecta el patrón: varias transacciones < 5€ en 2 minutos (prueba de tarjeta)
transactions
    .keyBy(Transaction::getCardNumber)
    .window(TumblingEventTimeWindows.of(Time.minutes(2)))
    .aggregate(new AggregateFunction<>() {
        public List<Transaction> add(Transaction tx, List<Transaction> acc) {
            if (tx.getAmount() < 5.0) acc.add(tx);
            return acc;
        }
        public CardTestAlert getResult(List<Transaction> acc) {
            if (acc.size() >= 3) return new CardTestAlert(acc);
            return null;
        }
    })
    .filter(alert -> alert != null)
    .addSink(fraudAlertSink);

2. Monitorización IoT Industrial

Las fábricas modernas tienen miles de sensores que envían datos continuamente: temperatura, presión, vibración, consumo eléctrico. Flink procesa estos streams para detectar anomalías y predecir fallos antes de que ocurran.

🏭
Caso real: Volkswagen usa Apache Flink para monitorizar sus líneas de producción en tiempo real. Procesan datos de más de 10.000 sensores simultáneamente y detectan problemas antes de que la pieza defectuosa llegue al siguiente proceso.
java — Flink IoT: detectar temperatura anómala y correlacionar con vibración 
DataStream<SensorReading> sensorStream = env
    .addSource(new MQTTSource("factory/sensors/#"));

// 1. Filtrar solo sensores de temperatura de zona A
DataStream<SensorReading> tempZoneA = sensorStream
    .filter(s -> s.getType().equals("TEMP") && s.getZone().equals("A"));

// 2. Media de temperatura por máquina en ventana de 30 segundos
DataStream<MachineStats> tempStats = tempZoneA
    .keyBy(SensorReading::getMachineId)
    .window(TumblingEventTimeWindows.of(Time.seconds(30)))
    .aggregate(new AvgTempAggregator());

// 3. Alerta si temperatura media supera 85°C
tempStats
    .filter(stats -> stats.getAvgTemp() > 85.0)
    .map(stats -> new CriticalAlert(
        stats.getMachineId(),
        "TEMPERATURA CRÍTICA: " + stats.getAvgTemp() + "°C",
        AlertLevel.CRITICAL
    ))
    .addSink(alertingSink);

3. Dashboard de Ventas en Tiempo Real (E-commerce)

Las grandes tiendas online necesitan dashboards actualizados en tiempo real: ventas por producto, categoría, región, comparación con el año anterior... Todo calculado mientras los usuarios compran.

🛒
Caso real: Durante el Black Friday de 2023, Amazon procesó más de 350.000 pedidos por hora. Sistemas basados en Flink actualizan en tiempo real los rankings de productos más vendidos, el inventario y las recomendaciones.
sql — Flink SQL: ranking de productos en tiempo real 
-- Tabla fuente: pedidos llegando de Kafka
CREATE TABLE orders (
    order_id    BIGINT,
    product_id  STRING,
    category    STRING,
    amount      DECIMAL(10, 2),
    quantity    INT,
    order_ts    TIMESTAMP(3),
    WATERMARK FOR order_ts AS order_ts - INTERVAL '10' SECOND
) WITH ('connector' = 'kafka', 'topic' = 'orders', 'format' = 'json');

-- Top 5 productos más vendidos en la última hora
SELECT product_id, SUM(quantity) AS total_units, SUM(amount) AS revenue
FROM orders
WHERE order_ts >= CURRENT_TIMESTAMP - INTERVAL '1' HOUR
GROUP BY product_id
ORDER BY total_units DESC
LIMIT 5;

-- Ventas por categoría en ventana tumbling de 5 minutos
SELECT
    category,
    TUMBLE_START(order_ts, INTERVAL '5' MINUTE) AS window_start,
    SUM(amount)   AS total_revenue,
    COUNT(*)      AS num_orders
FROM orders
GROUP BY category, TUMBLE(order_ts, INTERVAL '5' MINUTE);

4. Análisis de Logs en Tiempo Real

Plataformas como Netflix, Uber y Spotify generan terabytes de logs al día. Flink procesa estos logs en tiempo real para detectar errores, monitorizar el rendimiento y generar alertas.

java — Detectar pico de errores 500 en microservicios 
DataStream<LogEvent> logs = env
    .addSource(new FlinkKafkaConsumer("app-logs", schema, props));

// Contar errores 500 por servicio en ventanas de 1 minuto
logs
    .filter(log -> log.getStatusCode() == 500)
    .map(log -> Tuple2.of(log.getServiceName(), 1))
    .keyBy(t -> t.f0)
    .window(TumblingEventTimeWindows.of(Time.minutes(1)))
    .sum(1)
    .filter(t -> t.f1 > 100)   // más de 100 errores en 1 minuto
    .map(t -> new ServiceAlert(
        t.f0,
        "PICO DE ERRORES: " + t.f1 + " errores en el último minuto"
    ))
    .addSink(pagerDutySink);  // notificar al equipo de guardia

6. Resumen: ¿Cuándo elegir Flink?

Necesidad ¿Flink? Alternativa
Latencia < 100ms ✅ Ideal Solo Flink lo garantiza
Procesamiento stateful complejo ✅ Ideal Spark (con más código)
Event Time y eventos tardíos ✅ Ideal Spark Streaming (limitado)
Batch + ML + SQL familiar ⚠️ Posible Spark (más maduro para esto)
Latencia de segundos aceptable ⚠️ Posible Spark Streaming suficiente
Solo procesamiento batch histórico ❌ No ideal Spark, Hive, Trino
