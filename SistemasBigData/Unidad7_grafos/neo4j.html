Neo4j en la Práctica

Cómo se traduce todo lo anterior a Neo4j: el modelo de propiedades de grafo, el lenguaje Cypher y casos de uso reales en Big Data.

¿Qué es Neo4j?

Neo4j es una base de datos de grafos nativa. A diferencia de una base de datos SQL que guarda datos en tablas, Neo4j los guarda directamente como grafos: nodos, relaciones y propiedades.

La palabra "nativa" es importante: Neo4j no simula un grafo encima de tablas. Su motor está diseñado desde cero para navegar relaciones, lo que lo hace extremadamente eficiente para consultas que involucran muchas conexiones.

🏦 SQL (Base de datos relacional)
  • Datos en tablas con filas y columnas
  • Relaciones mediante JOINs
  • Eficiente para datos tabulares
  • Lento con muchos JOINs encadenados
🔷 Neo4j (Base de datos de grafos)
  • Datos como nodos y relaciones
  • Las relaciones son "ciudadanos de primera clase"
  • Eficiente para navegar conexiones
  • Ideal cuando las relaciones SON los datos

El Modelo de Propiedades de Grafo (LPG)

Neo4j usa el Labeled Property Graph (LPG). Esto significa que:

  • Los nodos pueden tener una o más etiquetas (ej: :Persona, :Producto)
  • Las relaciones tienen un único tipo (ej: :COMPRA, :CONOCE)
  • Tanto nodos como relaciones pueden tener propiedades (pares clave-valor)
  • Todas las relaciones tienen una dirección (aunque puedes ignorarla en las consultas)

El Lenguaje Cypher

Cypher es el lenguaje de consultas de Neo4j, diseñado para leer como un patrón visual. Usa flechas ASCII para representar el grafo que quieres encontrar o crear.

🎯
La idea clave de Cypher: "Dibuja el patrón que buscas y Neo4j lo encontrará." Si buscas personas que compraron el mismo producto, simplemente "dibujas" ese patrón en Cypher.

Sintaxis básica

cypher — Anatomía de una consulta 
// NODOS: Se representan con paréntesis ()
// (variable:Etiqueta {propiedad: valor})

(persona:Persona {nombre: "Ana"})
(p:Producto)
()   // nodo anónimo (cualquier nodo)

// RELACIONES: Se representan con corchetes []
// -[:TIPO {propiedad: valor}]->

-[:COMPRA]->           // relación dirigida
-[r:CONOCE]-           // sin dirección, con variable r
-[*1..3]->           // hasta 3 saltos (cualquier tipo)

CRUD básico: Crear, Leer, Actualizar, Eliminar

cypher — CREATE (Crear) 
// Crear nodos
CREATE (:Persona {nombre: "Ana", edad: 29})
CREATE (:Producto {nombre: "MacBook", precio: 1299})

// Crear relación entre nodos existentes
MATCH (a:Persona {nombre: "Ana"}),
      (p:Producto {nombre: "MacBook"})
CREATE (a)-[:COMPRA {fecha: "2024-01-15"}]->(p)
cypher — MATCH + RETURN (Leer) 
// Encontrar todos los productos que ha comprado Ana
MATCH (a:Persona {nombre: "Ana"})-[:COMPRA]->(p:Producto)
RETURN p.nombre, p.precio

// Encontrar amigos de amigos de Ana (2 saltos)
MATCH (a:Persona {nombre: "Ana"})-[:CONOCE*2]->(distante:Persona)
RETURN distante.nombre

// Contar cuántos productos tiene cada persona
MATCH (p:Persona)-[:COMPRA]->(prod:Producto)
RETURN p.nombre, count(prod) AS total
ORDER BY total DESC
cypher — SET y DELETE (Actualizar y Eliminar) 
// Actualizar una propiedad
MATCH (p:Persona {nombre: "Ana"})
SET p.edad = 30

// Eliminar un nodo y todas sus relaciones
MATCH (p:Persona {nombre: "Ana"})
DETACH DELETE p
// DETACH = elimina primero las relaciones

Casos de Uso con Ejemplos Completos

Caso 1: Red Social — ¿Quién tienen amigos en común con Ana?

cypher 
MATCH (ana:Persona {nombre: "Ana"})-[:AMIGO]->(amigo:Persona)
                              <-[:AMIGO]-(otro:Persona)
WHERE otro.nombre <> "Ana"  // excluir a Ana misma
  AND NOT (ana)-[:AMIGO]-(otro)  // que no sea ya amiga
RETURN otro.nombre AS sugerencia,
       count(amigo) AS amigos_en_comun
ORDER BY amigos_en_comun DESC
LIMIT 5  // Top 5 sugerencias de amistad

Caso 2: E-commerce — Sistema de recomendación

cypher 
// "Los que compraron X también compraron Y"
MATCH (yo:Persona {nombre: "Ana"})-[:COMPRA]->(p:Producto)
                              <-[:COMPRA]-(otro:Persona)
                              -[:COMPRA]->(recom:Producto)
WHERE NOT (yo)-[:COMPRA]->(recom)  // que Ana no lo haya comprado ya
RETURN recom.nombre AS recomendacion,
       count(*) AS popularidad
ORDER BY popularidad DESC
LIMIT 10

Caso 3: Detección de Fraude

cypher 
// Detectar cuentas que comparten el mismo teléfono (posible fraude)
MATCH (c1:Cuenta)-[:USA_TELEFONO]->(t:Telefono)
                  <-[:USA_TELEFONO]-(c2:Cuenta)
WHERE c1 <> c2  // que sean cuentas diferentes
RETURN t.numero, 
       collect(c1.id) AS cuentas_sospechosas,
       count(*) AS num_cuentas
ORDER BY num_cuentas DESC

Caso 4: Rutas — Camino más corto entre ciudades

cypher 
// Camino más corto (sin pesos)
MATCH (origen:Ciudad {nombre: "Madrid"}),
      (destino:Ciudad {nombre: "Alicante"})
MATCH ruta = shortestPath((origen)-[:CONECTA*]-(destino))
RETURN [nodo IN nodes(ruta) | nodo.nombre] AS paradas,
       length(ruta) AS num_conexiones

MERGE: el Upsert de Neo4j

MERGE es uno de los comandos más poderosos de Cypher. Funciona como un "crea si no existe, encuentra si existe". Es perfecto para cargar datos sin crear duplicados.

cypher — MERGE 
// Si la persona existe, la encuentra. Si no, la crea.
MERGE (p:Persona {email: "ana@ejemplo.com"})
ON CREATE SET p.nombre = "Ana", p.creado = timestamp()
ON MATCH SET  p.ultimo_acceso = timestamp()

// Ideal para cargas masivas de datos en Big Data:
// Procesas millones de registros sin preocuparte de duplicados

Neo4j en el Ecosistema Big Data

🔄

Ingestión desde Spark

Puedes usar el conector Neo4j Spark Connector para cargar datos desde PySpark directamente a Neo4j con DataFrames.

Graph Data Science (GDS)

🧠

Graph ML

GDS incluye Graph Neural Networks y embeddings de grafos para usar los grafos como features en modelos ML.

Carga de CSV

API REST / Bolt

Neo4j AuraDB

La versión cloud de Neo4j. Cero configuración, escalado automático. Ideal para proyectos Big Data en la nube.

Ejemplo Completo: Carga de Datos CSV

Un flujo típico para cargar datos de un proyecto Big Data en Neo4j:

cypher — LOAD CSV con MERGE 
// Paso 1: Crear índice para velocidad
CREATE INDEX persona_email IF NOT EXISTS
FOR (p:Persona) ON (p.email)

// Paso 2: Cargar usuarios desde CSV
LOAD CSV WITH HEADERS FROM 'file:///usuarios.csv' AS row
MERGE (p:Persona {email: row.email})
SET p.nombre = row.nombre,
    p.edad = toInteger(row.edad),
    p.ciudad = row.ciudad

// Paso 3: Cargar relaciones de amistad
LOAD CSV WITH HEADERS FROM 'file:///amistades.csv' AS row
MATCH (a:Persona {email: row.email_a})
MATCH (b:Persona {email: row.email_b})
MERGE (a)-[:AMIGO {desde: row.fecha}]-(b)
