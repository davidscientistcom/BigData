Módulo 2: Cypher — Nivel Básico

El lenguaje que convierte patrones visuales en consultas. Si puedes dibujarlo en una pizarra, puedes consultarlo.

1. Sintaxis ASCII del grafo

Cypher refleja visualmente la estructura del grafo. La notación es intuitiva por diseño:

SímboloRepresentaEjemplo
()Nodo (círculo vacío)(n) — nodo sin label
(:Label)Nodo con etiqueta(:Persona)
(n:Label)Nodo con variable y etiqueta(p:Persona) — puedes reusar p
{prop: val}Propiedades (filtro o asignación)({nombre: "Ana"})
-[]->Relación dirigida-[:CONOCE]->
-[]-Relación sin dirección (match ambos sentidos)-[:AMIGO]-
[r:TIPO]Relación con variable[r:COMPRA]
[*1..3]Variable de longitud (1 a 3 saltos)-[*1..3]->
cypher — Anatomía completa de un patrón
// Nodo completo con variable, label y propiedades
(p:Persona:Investigador {nombre: "Deivit", activo: true})

// Relación completa con variable, tipo y propiedades
(p)-[r:PUBLICÓ {año: 2024, citaciones: 150}]->(a:Paper)

// Un nodo puede tener múltiples labels
(x:Sensor:IoT:Activo {id: "SNS-001", tipo: "LIDAR"})

// Patrón complejo en una sola expresión
(v:Vehiculo)-[:TIENE]->(s:Sensor)-[:DETECTÓ {timestamp: datetime()}]->(o:Obstáculo)

⚠️
Null en Cypher: Null no es igual a null. null = null evalúa a null, no a true. Para comprobar nulos, usa siempre IS NULL o IS NOT NULL.

3. CREATE — Construir el grafo

cypher — Ejemplos CREATE variados
-- E1: Nodo simple
CREATE (u:User {nombre: "Ana", email: "ana@lab.es"})

-- E2: Varios nodos en una sola sentencia
CREATE
  (v1:Vehiculo {id: "AV-001", modelo: "Tesla Model S"}),
  (v2:Vehiculo {id: "AV-002", modelo: "Waymo"}),
  (s1:Sensor {tipo: "LIDAR", frecuencia: 10})

-- E3: Nodo con lista como propiedad
CREATE (p:Paper {
  titulo: "Graph Neural Networks for Autonomous Driving",
  autores: ["Deivit", "Merino"],
  keywords: ["GNN", "autonomous", "safety"],
  publicado: date("2025-09-15"),
  citaciones: 45
})

-- E4: Nodo + relación en una sola sentencia
CREATE (inv:Investigador {nombre: "Deivit"})-[:PUBLICÓ {año: 2025}]->(p:Paper {titulo: "GNN Survey"})

-- E5: MATCH primero, luego CREATE la relación (correcto)
MATCH (a:User {nombre: "Ana"}), (b:User {nombre: "Bob"})
CREATE (a)-[:COLABORA_CON {proyectos: 3, desde: date()}]->(b)

-- E6: Crear nodo de fraude bancario con propiedades complejas
CREATE (t:Transaccion {
  id: "TXN-98123",
  importe: 4500.75,
  moneda: "EUR",
  timestamp: datetime("2026-03-25T23:45:00Z"),
  ip_origen: "192.168.1.1",
  sospechosa: true
})
⚠️
Antipatrón: CREATE siempre crea, incluso si el nodo ya existe. Si ejecutas el mismo CREATE dos veces, tendrás duplicados. Para evitarlo, usa MERGE (Módulo 3). En producción, casi nunca usarás CREATE puro para entidades — solo para relaciones entre nodos ya existentes.

4. MATCH — Navegar el grafo

cypher — Ejemplos MATCH de menos a más complejo
-- E1: Todos los nodos con un label
MATCH (p:Paper) RETURN p.titulo

-- E2: Filtrar por propiedad directamente en el patrón
MATCH (u:User {activo: true}) RETURN u.nombre, u.email

-- E3: Relación simple — quién publicó qué paper
MATCH (inv:Investigador)-[:PUBLICÓ]->(p:Paper)
RETURN inv.nombre, p.titulo, p.publicado

-- E4: Sin dirección — todos los colaboradores (en cualquier sentido)
MATCH (a:User)-[:COLABORA_CON]-(b:User)
RETURN a.nombre, b.nombre

-- E5: Múltiples saltos — coautores de coautores (red académica)
MATCH (yo:Investigador {nombre: "Deivit"})-[:PUBLICÓ]->(p:Paper)<-[:PUBLICÓ]-(coautor:Investigador)
RETURN coautor.nombre, count(p) AS papers_juntos
ORDER BY papers_juntos DESC

-- E6: Multiple labels en el patrón
MATCH (v:Vehiculo)-[:TIENE]->(s:Sensor)
WHERE s.tipo IN ["LIDAR", "RADAR"]
RETURN v.id, collect(s.tipo) AS sensores

-- E7: Patrón de fraude — cuenta conectada a 3+ cuentas sospechosas
MATCH (c:Cuenta)-[:TRANSACCIÓN_A]->(destino:Cuenta {sospechosa: true})
WITH c, count(destino) AS cuentas_sospechosas
WHERE cuentas_sospechosas >= 3
RETURN c.id, cuentas_sospechosas
💡
Regla de oro: Siempre incluye al menos un label en cada nodo del MATCH. Sin label, Neo4j hace un AllNodesScan y escanea toda la base de datos. Con label, usa el Token Lookup Index — instantáneo.

5. WHERE — Filtrado avanzado

cypher — Todo el arsenal de WHERE
-- Comparadores básicos
MATCH (t:Transaccion) WHERE t.importe > 1000 RETURN t.id

-- AND / OR / NOT / XOR
MATCH (u:User)
WHERE (u.rol = "admin" OR u.rol = "superuser")
  AND NOT u.bloqueado
RETURN u.nombre

-- IS NULL / IS NOT NULL
MATCH (p:Paper) WHERE p.doi IS NULL RETURN p.titulo -- papers sin DOI

-- IN (lista literal)
MATCH (s:Sensor) WHERE s.tipo IN ["LIDAR", "RADAR", "CAMERA"] RETURN s

-- STARTS WITH / ENDS WITH / CONTAINS (string matching)
MATCH (u:User) WHERE u.email ENDS WITH "@ua.es" RETURN u.nombre
MATCH (p:Paper) WHERE p.titulo CONTAINS "autonomous" RETURN p.titulo

-- Expresión regular con =~
MATCH (u:User) WHERE u.telefono =~ "\\+34.*" RETURN u.nombre

-- WHERE con patrón de grafo (existencia de relación)
MATCH (p:Persona)
WHERE (p)-[:PERTENECE]->(:Grupo {nombre: "IA Lab"})
RETURN p.nombre

-- WHERE NOT EXISTS (filtrar si NO tiene relación)
MATCH (u:User)
WHERE NOT (u)-[:TIENE_ROL]->(:Rol)
RETURN u.nombre -- usuarios sin ningún rol asignado

-- Comparación de fechas
MATCH (t:Transaccion)
WHERE t.timestamp > datetime("2026-01-01T00:00:00Z")
  AND t.importe > 5000
RETURN t.id, t.importe, t.timestamp

6. RETURN — Proyectar resultados

cypher — RETURN avanzado
-- E1: Alias con AS
MATCH (p:Paper) RETURN p.titulo AS titulo, p.citaciones AS citas

-- E2: DISTINCT — eliminar duplicados
MATCH (u:User)-[:COMPRÓ]->(p:Producto)-[:DE_CATEGORÍA]->(c:Categoría)
RETURN DISTINCT c.nombre -- categorías únicas compradas

-- E3: ORDER BY con múltiples criterios
MATCH (p:Paper)
RETURN p.titulo, p.citaciones, p.publicado
ORDER BY p.citaciones DESC, p.publicado DESC

-- E4: SKIP y LIMIT (paginación)
MATCH (u:User) RETURN u.nombre SKIP 20 LIMIT 10  -- página 3

-- E5: Retornar el nodo completo, solo propiedades específicas, o un map
MATCH (v:Vehiculo)
RETURN
  v,                         -- nodo entero
  v.id,                      -- propiedad
  {id: v.id, modelo: v.modelo} AS resumen  -- map

-- E6: Retornar conteo y agregación
MATCH (inv:Investigador)-[:PUBLICÓ]->(p:Paper)
RETURN inv.nombre, count(p) AS total, sum(p.citaciones) AS total_citas
ORDER BY total_citas DESC

7. SET y REMOVE — Modificar datos

cypher — SET y REMOVE exhaustivos
-- SET propiedad simple
MATCH (u:User {email: "ana@lab.es"})
SET u.activo = true
RETURN u

-- SET multiple propiedades a la vez
MATCH (v:Vehiculo {id: "AV-001"})
SET v.velocidad_max = 120, v.nivel_autonomia = 4, v.actualizado = datetime()

-- SET con += (merge de propiedades, no sobreescribe las existentes)
MATCH (p:Persona {id: "P001"})
SET p += {telefono: "+34600123456", ciudad: "Alicante"}

-- Añadir un nuevo label
MATCH (u:User {rol: "admin"})
SET u:Admin

-- SET propiedad en relación
MATCH (a:User)-[r:SIGUE]->(b:User)
WHERE a.nombre = "Ana" AND b.nombre = "Bob"
SET r.notificaciones = true

-- REMOVE propiedad
MATCH (u:User {email: "ana@lab.es"})
REMOVE u.token_temporal

-- REMOVE label
MATCH (u:User:Admin {email: "ana@lab.es"})
REMOVE u:Admin  -- quitar label Admin, sigue siendo User

8. DELETE y DETACH DELETE

cypher — Borrado seguro
-- Borrar relación (primero match explícito)
MATCH (a:User {nombre: "Ana"})-[r:SIGUE]->(b:User)
DELETE r

-- ERROR: no puedes borrar un nodo con relaciones activas
-- MATCH (u:User {nombre: "Ana"}) DELETE u  ← Neo.ClientError.Schema.ConstraintViolation

-- DETACH DELETE: borra nodo Y todas sus relaciones (hazlo con cuidado)
MATCH (u:User {nombre: "Temporal"})
DETACH DELETE u

-- Borrar todos los nodos y relaciones de la DB (solo desarrollo)
MATCH (n) DETACH DELETE n
⚠️
DETACH DELETE en producción: Nunca ejecutes MATCH (n) DETACH DELETE n en una base de datos de producción. No hay rollback automático. Siempre trabaja con transacciones explícitas y verifica primero con MATCH (n) RETURN count(n).

9. Propiedades en Relaciones

A diferencia de SQL (donde necesitas una tabla intermedia), en Neo4j las relaciones son ciudadanos de primera clase y pueden tener propiedades propias.

cypher — Propiedades en relaciones
-- Crear relación con propiedades (red de vuelos)
MATCH (mad:Aeropuerto {iata: "MAD"}), (bcn:Aeropuerto {iata: "BCN"})
CREATE (mad)-[:VUELO {
  compañia: "Iberia",
  duracion_min: 75,
  precio_medio: 89.99,
  frecuencia_diaria: 12
}]->(bcn)

-- Consultar propiedades de relaciones
MATCH (origen:Aeropuerto)-[v:VUELO]->(destino:Aeropuerto)
WHERE v.precio_medio < 100
RETURN origen.iata, destino.iata, v.compañia, v.precio_medio
ORDER BY v.precio_medio

-- Red de colaboración científica — cuántos papers juntos
MATCH (a:Investigador)-[c:COLABORA_CON]->(b:Investigador)
WHERE c.papers_juntos >= 3
RETURN a.nombre, b.nombre, c.papers_juntos
ORDER BY c.papers_juntos DESC

-- E-commerce: un usuario compró un producto — cuándo y a qué precio
MATCH (u:User)-[c:COMPRÓ]->(p:Producto)
RETURN u.nombre, p.nombre, c.precio_compra, c.fecha, c.cantidad

🧪 Ejercicios — Módulo 2

📝
E1 (Básico): Crea un grafo de tu equipo de investigación: al menos 3 investigadores, 3 papers y las relaciones [:PUBLICÓ]. Uno de los papers debe tener 2 autores. Carga con el dataset movies de :play movies para practicar antes.
📝
E2 (Básico): Con el dataset movies, encuentra todas las películas con Tom Hanks. Retorna el título y el año. Ordena por año descendente. Solución: MATCH (tom:Person {name:"Tom Hanks"})-[:ACTED_IN]->(m:Movie) RETURN m.title, m.released ORDER BY m.released DESC
📝
E3 (Medio): Escribe una query que encuentre todos los usuarios que cumplen: (a) su email termina en "@ua.es" OR (b) tienen propiedad admin: true. Usa WHERE con operadores lógicos. Devuelve solo los distintos emails.
📝
E4 (Medio): Crea un grafo de transacciones bancarias sospechosas: 5 cuentas, 8 transacciones con propiedades (importe, timestamp). Identifica las cuentas que han enviado y recibido dinero (están en ambos extremos de una transacción).
📝
E5 (Avanzado): Usando el operador =~ y regex, encuentra todos los papers cuyos títulos contienen una palabra que empieza por mayúscula seguida de al menos 4 letras minúsculas. Explica por qué esta query puede ser lenta y cómo lo resolverías en producción (pista: full-text index).
